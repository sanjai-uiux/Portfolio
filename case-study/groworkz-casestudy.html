01 — Overview

A Complete Enterprise Tool, Redesigned For Your Pocket.

My assignment was to design the complete mobile version of the platform, adapting every major module to a thumb-friendly, context-aware mobile experience without losing the depth of functionality that enterprise users depend on.

02 — Problem Statement

Enterprise Depth Doesn't Fit A 390px Screen.
Or Does It?Yes, It Does.

The Groworkz web platform is feature-dense by design. Side navigation with 20+ menu items, modal dialogs for every form, data tables with pagination — all built for large displays and mouse interaction.

The business need was clear: field teams, managers, and employees needed to access and action tasks, raise requests, check attendance, and respond to tickets while away from their desks. The existing web UI simply didn't translate.

"How Do You Preserve The Full Capability Of a 20-Module Enterprise SaaS Platform When All You Have Is A 390 × 844 Screen And A Thumb?"

Information overload Navigation complexity Modal + page conversion Form density on small screens Approval workflows on mobile Real-time time tracking

07 — Key Design Decisions

The Choices That Shaped The Experience.

Bottom Nav Over Sidebar

The web's collapsible sidebar with 20+ nested items couldn't work on mobile. Replaced it with a 5-item persistent bottom tab bar with an in-section navigation pattern — keeping primary modules always one tap away.

Modals → Full Screens

Every web form was inside a modal dialog. On mobile, these became full-screen pages with a back arrow header. This follows iOS/Android patterns, supports the soft keyboard better, and allows scrolling through long forms without viewport constraints of a floating overlay.

Bottom Sheets For Filters

Filter controls on web used inline dropdowns and sidebars. On mobile, all filtering is done through bottom sheets — half-screen drawers that appear on demand. This keeps the list view uncluttered and gives users enough space for multi-select filter combinations.

Attendance Grid Redesign

The web's dense 31-column attendance matrix (showing all 35 employees across an entire month) was completely rethought for mobile. Three progressive views — Today list, This Week strip, This Month compact grid — each serve different user needs without data loss.

In-App Notifications

The web's notification bell dropdown was translated to a push-notification-style list screen with Unread/Read segmentation. Each notification links directly to the relevant task or record, supporting the "deep link" behavior mobile users expect.

Persistent Task Timer

The task timer, pinned in the modal footer on web, became a persistent timer bar at the screen bottom — always visible across screens during an active time log. Mirrors the always-visible web timer while respecting mobile navigation space.

11 — Learnings & Reflections

What This Project Taught Me.

01
Adaptation ≠ Shrinking
Designing for mobile isn't about making the web version smaller. It requires rethinking interaction paradigms entirely — what works as a hover action on desktop needs to become a swipe, tap, or bottom sheet on mobile.
02
Component systems pay dividends
Investing early in a consistent component library — especially info cards, bottom sheets, and form controls — meant I could design later modules much faster and maintain visual consistency across 100+ screens.
03
Complexity can be progressive
The attendance redesign showed that progressive disclosure works for complex data too. Showing the right amount of information at each zoom level (today → week → month) respects both the user's attention and the screen's limitations.
